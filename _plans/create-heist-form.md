# Plan: Create Heist Form

## Context

The `/heists/create` page is a stub with just a title. This plan builds a `CreateHeistForm` component that lets authenticated users write a new heist document to Firestore, picking an assignee from the existing operatives list fetched from the `users` collection. `createdAt`, `deadline`, and `finalStatus` are set programmatically. On success the user is redirected to `/heists`.

---

## Files to Create / Modify

### `types/heist.ts` — add `CreateHeistInput`

Add the `CreateHeistInput` interface (per the `skills/firestore-schemas` convention):

- Excludes `id` (auto-generated by Firestore `addDoc`)
- `createdAt: FieldValue` — use `serverTimestamp()`
- `deadline: Timestamp` — set client-side to 48 h from now
- `finalStatus: null` — always `null` on creation
- All other fields match `Heist` minus `id`

Add `import { FieldValue } from 'firebase/firestore'` (alongside existing `Timestamp`).

### `components/CreateHeistForm/CreateHeistForm.tsx` _(new)_

**`'use client'` directive required.**

**State:** `title`, `description`, `assignedTo`, `loading`, `error`, `operatives`, `operativesLoading`

**On mount:** `getDocs(collection(db, 'users'))` → map to `{ uid: doc.id, codename: doc.data().codename }[]`

**`handleSubmit`:** build `CreateHeistInput`, `addDoc(collection(db, 'heists'), data)`, `router.push('/heists')` on success
